# Política de Segurança — APACApp Oncologia

## Versões suportadas

| Versão | Suporte de segurança |
| ------ | -------------------- |
| `main` (latest) | ✅ Recebe correções |
| versões antigas | ❌ Sem suporte |

## Reportando uma vulnerabilidade

**Não abra uma _issue_ pública** para reportar vulnerabilidades de segurança.

Use o canal privado do GitHub:
👉 **[Reportar vulnerabilidade em privado](https://github.com/silasjuba-del/apac-oncologia/security/advisories/new)**

### O que incluir no relatório

1. Descrição clara do problema e do impacto potencial
2. Passos para reproduzir (PoC, se possível)
3. Versão/commit afetado
4. Sugestão de correção (opcional)

### Tempo de resposta esperado

| Etapa | Prazo |
| ----- | ----- |
| Confirmação de recebimento | até **3 dias úteis** |
| Avaliação de severidade | até **7 dias úteis** |
| Publicação do patch | depende da severidade (crítica: < 14 dias) |

## Escopo

Este repositório contém uma aplicação clínica (APACs oncológicas). Dados de pacientes **nunca** devem ser incluídos em relatórios de bug ou issues.

## Contato

- GitHub Security Advisories (preferencial)
- E-mail do mantenedor (disponível no perfil GitHub)
